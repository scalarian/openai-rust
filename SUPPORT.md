# Support

Use the repository entry point that matches the job:

- Bugs or regressions: open a bug report with a minimal reproduction and the crate version or commit SHA.
- API coverage or ergonomics requests: open a feature request and explain the use case.
- Documentation problems: open a docs issue with the broken page, example, or command.
- Security concerns: do not file a public issue. Follow [SECURITY.md](SECURITY.md).

Before opening an issue, check:

- [README.md](README.md) for the current public project overview.
- [docs/quickstart.md](docs/quickstart.md) for first-use commands.
- [docs/migration-guide.md](docs/migration-guide.md) if you are moving from chat-completions or legacy completions flows.
- [docs/api-coverage.md](docs/api-coverage.md) and [docs/intentional-gaps.md](docs/intentional-gaps.md) for supported and intentionally deferred surfaces.

This project does not offer commercial support or response-time guarantees.
